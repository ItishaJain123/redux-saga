Redux Saga – Cheat Sheet

This page summarizes the most important Redux Saga concepts for quick revision.

Core concepts

  • Saga is Redux middleware
  • Uses generator functions (function*)
  • Handles side effects only
  • Reducers remain pure
  • Logic reacts to actions, not promises

Watcher vs Worker

  • Watcher listens for actions
  • Worker performs async work
  • Improves separation of concerns

Common effects

  • call(fn) – call async function
  • put(action) – dispatch action
  • take(action) – wait for action
  • takeEvery – run on every action
  • takeLatest – cancel previous task
  • select – read from store
  • delay – pause execution
  • race – run effects in parallel
  • cancel – stop a running task

Error handling

  • Use try / catch inside sagas
  • Dispatch failure actions
  • Keep error logic out of components

Testing

  • Sagas are generator functions
  • Effects are plain objects
  • Test by stepping through next()
  • No real API calls needed

When to use Redux Saga

  • complex async workflows
  • cancellation or retries needed
  • background tasks or polling
  • large and long-lived applications

Key takeaway

Redux Saga brings structure, control, and testability to complex asynchronous logic.
