Effects in Redux Saga

In Redux Saga, asynchronous work is handled using effects.

An effect is a plain JavaScript object that describes what should happen, not how it should happen.

Why effects exist

Instead of calling async functions directly, Redux Saga lets you yield effects.

  • execution stays under Sagaâ€™s control
  • flows can be paused or cancelled
  • logic becomes easy to test

Commonly used effects

call

Used to call an asynchronous function such as an API request.

yield call(fetchUsersApi);

Saga waits until the function resolves before moving to the next step.

put

Used to dispatch a Redux action.

yield put({ type: "USERS_FETCH_SUCCESS", payload: users });

This is similar to dispatch, but controlled by Saga.

take

Used to pause the saga until a specific action is dispatched.

yield take("FETCH_USERS");

select

Used to read data from the Redux store.

const userId = yield select(state => state.user.id);

Declarative nature of effects

When you write:

yield call(fetchData);

You are not executing the function immediately. You are describing the intent.

Redux Saga decides when and how to execute it.

Key takeaway

Effects are the foundation of Redux Saga. They make async flows predictable, controllable, and easy to reason about.
