Redux Saga Effects

Redux Saga effects are instructions that a saga gives to the Redux-Saga middleware. The saga itself does not execute async code. Instead, it describes what should happen, and Redux-Saga takes care of executing it.

call

The call effect is used when you want to run a function and wait for its result. This is most commonly used for API calls. When a saga yields a call effect, it pauses until the function finishes and then continues with the returned data.

const data = yield call(fetchUsersApi);

put

The put effect is used to dispatch an action to the Redux store. It works like dispatch, but it is controlled by Redux Saga. Reducers handle the action and update the state as usual.

yield put({ type: "FETCH_SUCCESS", payload: data });

take

The take effect pauses the saga until a specific action is dispatched. Once the action is received, the saga continues execution. It listens only once and then stops.

yield take("LOGIN");

takeEvery

The takeEvery effect listens for an action and runs a worker saga every time that action is dispatched. It does not cancel any previous task. Multiple worker sagas can run at the same time.

yield takeEvery("FETCH_USERS", fetchUsersWorker);

takeLatest

The takeLatest effect is similar to takeEvery, but it cancels the previous running task and keeps only the latest one. This is very useful for search boxes, filters, and fast user interactions.

yield takeLatest("SEARCH_USERS", searchUsersWorker);

fork

The fork effect starts a task in the background. The saga does not wait for it to finish and continues execution immediately. This is used for background processes like polling or syncing.

const task = yield fork(backgroundSyncSaga);

cancel

The cancel effect is used to stop a running background task that was started using fork. This is useful when a user navigates away or when a task is no longer needed.

yield cancel(task);

select

The select effect allows a saga to read data from the Redux store. It does not modify the state, it only reads the current value.

const userId = yield select(state => state.user.id);

delay

The delay effect pauses the saga for a specific amount of time. It is often used for polling, retry logic, or showing loaders.

yield delay(1000);

race

The race effect runs multiple effects at the same time and continues with the one that finishes first. All other running effects are automatically cancelled. This is commonly used to implement timeouts.

yield race({ response: call(fetchData), timeout: delay(5000), });

all

The all effect is used to run multiple effects in parallel. The saga waits until all of them complete before continuing.

yield all([ call(fetchUsers), call(fetchPosts), ]);

Key takeaway

Redux Saga effects make asynchronous logic predictable and easy to control. They keep API calls, delays, and background work outside components, making the application easier to maintain and test.
