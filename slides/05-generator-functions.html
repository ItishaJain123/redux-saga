Generator Functions

Redux Saga is built on top of JavaScript generator functions.

At first, generator functions may look confusing, but the idea behind them is simple.

What is a generator function?

A generator function is a special type of function that can pause and resume its execution.

It is defined using function* and uses the yield keyword. 

  


function* exampleGenerator() {
  yield 1;
  yield 2;
  yield 3;
}

Each yield pauses the function and returns a value.

Why normal functions are not enough

Async logic often needs:

  • waiting for an API response
  • pausing execution
  • resuming after completion
  • handling success or failure

Promises handle this, but they execute immediately and are harder to control once started.

Why Redux Saga uses generators

Generators allow Redux Saga to:

  • pause execution at any step
  • resume when an effect completes
  • cancel running tasks
  • control complex async flows

Instead of running async code directly, Sagas describe each step using yield.

Generator in a Saga

  


function* fetchUserSaga() {
  yield call(fetchUserApi);
  yield put({ type: "USER_FETCH_SUCCESS" });
}

Here, each line runs step by step under Saga's control.

Key takeaway

Generator functions give Redux Saga its power. They make async flows readable, controllable, and easy to reason about.
